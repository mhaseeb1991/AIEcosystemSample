# AI Naming Convention

This document defines the naming standards for files within the `ai/` directory to maintain a clear visual hierarchy.

## 1. Core Rules & Templates
**Convention:** `SCREAMING_SNAKE_CASE.md`

Use this for high-level documentation and project-wide policies ("How we work").

- **Examples:** `FEATURE_TEMPLATE.md`, `TEST_GUIDELINES.md`, `DIRECTORY_RULES.md`

## 2. Feature Specifications & Agents
**Convention:** `snake_case.md`

Use this for feature files ("What we built") and AI agent role definitions.

- **Examples:** `user_profile.md`, `designer.md`, `testing_agent.md`

---

### Visual Hierarchy
- **UPPERCASE:** Guidelines and Templates (Authority)
- **lowercase:** Implementation and Roles (Content)
