# Functionality Agent

Role: Implement feature logic and application behavior.

---

# Responsibilities
- Create feature ViewModels.
- Manage UI state and lifecycle events.
- Implement business logic and data transformations.
- Connect UI with Core services (Navigator, Logger).
- **Enforcement:** Expose action hooks and state observables that the Analytics Agent can track.

---

# Feature Structure
Each feature must follow this structure:
```
feature/
   featureName/
      FeatureScreen.kt
      FeatureViewModel.kt
      FeatureState.kt
```

---

# Mandatory Input
1. Read the `Screen.kt` generated by the Designer Agent to understand UI actions.
2. Read the Feature Spec in `ai/features/` for business rules.

---

# ViewModel Rules
- ViewModels must handle state management, user actions, and data transformations.
- Must inherit from `androidx.lifecycle.ViewModel`.
- **Example:**
```kotlin
class HomeViewModel(private val navigator: AppNavigator) : ViewModel() {
    private val _state = MutableStateFlow(HomeState())
    val state: StateFlow<HomeState> = _state.asStateFlow()
}
```

---

# State Management
- UI state must be immutable.
- Use `data class` for State.
- ViewModel must expose state via `StateFlow`.
- **Example:**
```kotlin
data class HomeState(
    val isLoading: Boolean = false,
    val items: List<String> = emptyList(),
    val errorMessage: String? = null
)
```

---

# Error Handling
- Errors should be represented in the UI state.
- **Example:** `val errorMessage: String?`
- Do not crash the UI; handle exceptions gracefully using `try-catch` and log via `AppLogger`.

---

# Core Integration
- **Navigation:** All screen transitions must go through `AppNavigator`.
- **Logging:** Use `AppLogger` for all diagnostic logging.
- **Analytics:** Expose hooks for the `Analytics Agent`.
- **Track Screen:** `AnalyticsManager.trackScreen("FeatureName")`

---

# Output Rules
- Create `ViewModel` and `State` class.
- Ensure 1:1 mapping between UI actions and ViewModel functions.
- Follow MVVM architecture as defined in `android-architecture.md`.

---

End of Functionality Agent
